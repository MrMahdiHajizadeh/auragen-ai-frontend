Build a state-of-the-art, premium AI Avatar and Image Generation Web Application named "AuraGen AI". 

Use React, Tailwind CSS, Lucide Icons, and Framer Motion for smooth micro-animations. The application must look like a high-end, multi-million-dollar AI SaaS platform.

==================================================
1. DESIGN SYSTEM & VISUAL GUIDELINES
==================================================
- Theme: Sleek premium dark mode. Use deep space-gray (#05070c to #0b0f19) as the background.
- Accents: Glowing neon colors. Electric Indigo (#6366F1) for primary actions, Digital Cyan (#06B6D4) for information/progress, and Warm Amber (#F59E0B) for coins and premium statuses.
- Aesthetics: Ultra-glassmorphism (backdrop-filter: blur(16px), border: 1px solid rgba(255,255,255,0.06), translucent slate backgrounds).
- Radians: Subtle glowing radial gradients floating in the corners of the background.
- Animations: Active hover transitions (scale cards by translateY(-6px) with cubic-bezier curves), loading skeletons, and soft glowing outline transitions.

==================================================

==================================================
4. EXACT DJANGO/FASTAPI BACKEND BINDINGS DATA
==================================================
Ensure all React hooks, state objects, and form inputs strictly map to the keys defined in this Swagger API schema. You should build a smart API client that fetches from this schema but falls back to mock storage if the server is offline:

- Banners List Schema:
  Endpoint: GET `/api/banners/` -> Returns array of:
  `{ id: "UUID", name: "string", link: "URL", banner_url: "URL", description: "string" }`

- Categories List Schema:
  Endpoint: GET `/api/categories/` -> Returns array of:
  `{ id: "UUID", name: "string", color: "Hex Color", photo_url: "URL" }`

- Generators List Schema:
  Endpoint: GET `/api/generators/` -> Returns array of:
  `{ id: "UUID", name: "string", model_id: "string", required_images_count: integer, token_cost: integer, photo_url: "URL", color: "Hex", description: "string" }`

- Image Generation Trigger Schema:
  Endpoint: POST `/api/generate/{generator_id}/`
  Body: `{ input_image_ids: ["UUID_1", "UUID_2"], description: "string" }`
  Response: `{ session_id: "string", ws_url: "WebSocket URL for progress", message: "Accepted" }`

- Generated Gallery List Schema:
  Endpoint: GET `/api/generated/` -> Returns array of:
  `{ id: "UUID", output_image_url: "URL", generation_time: float, input_tokens: integer, output_tokens: integer, cost: float, generated_at: "date-time" }`

- Pricing Subscription Plans Schema:
  Endpoint: GET `/api/plans/` -> Returns array of:
  `{ id: "UUID", name: "string", coins: integer, price: integer (in Rials), bazaar_product_id: "string" }`

- Café Bazaar Purchase Verification Schema:
  Endpoint: POST `/api/plans/verify/`
  Body: `{ package_name: "string", product_id: "string", purchase_token: "string", plan_id: "UUID" }`
  Response: `{ success: true, message: "Purchase verified" }`
